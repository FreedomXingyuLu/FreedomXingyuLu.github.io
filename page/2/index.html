在网上查到当h总和一定时，c越大，结果越大。

通过01背包计算出每个C能得到的最大的H，复杂度∑C*N

#include <stdio.h>
#include <string.h>
#define NUM 50000
long long max(long long a,long long b)
{
 return a>b?a:b;
}
int main()
{
 long long n,N,i,j;
 long long dp[NUM],h[NUM],c[NUM];

scanf("%lld",&N);
 for(;N>0;N--)
 {
 memset(dp,0,sizeof(dp));
 scanf("%lld",&n);
 for(i=1;i<=n;i++)
 {
 scanf("%lld%lld",&h[i],&c[i]);
 for(j=NUM-1;j>=c[i];j--)
 {
 dp[j]=max(dp[j],dp[j-c[i]]+h[i]);
 }
 }
 long long ans=0;
 for(i=0;i<NUM;i++)
 ans=max(ans,dp[i]*dp[i]-dp[i]*i-i*i);
 printf("%lld\n",ans);
 }
 return 0;
}

最大平均值问题(USACO2003 March Open)

[题意简述]

读入一列正数，a1,a2,…,aN，以及一个数F。定义，

求Max{ave(a,b),1≤a,b≤N，且a≤b-F+1}，即求一段长度大于等于F且平均值最大的子串。

范围：F≤N≤105。

[分析]

枚举肯定不行。但可以使用图形化。

令前缀和

如果将S函数绘在平面直角坐标系内，这就是过点和点直线的斜率！

于是问题转化为：平面上已知N+1个点，，0≤i≤N，求横向距离大于等于F的任意两点连线的最大斜率。

数据如下时

即求之中的最大斜率

[构造下凸折线]

有序化一下，规定对i<j，只检查Pj向Pi的连线，对Pi不检查与Pj的连线。也就是说对任意一点，仅检查该点与在其前方的点的斜率。

当我们考察一个点Pt时，朴素的平方级算法依次选取t点之前的每一个被检查点p，考察直线p->Pt的斜率。但仔细观察，若集合内存在三个点Pi,Pj,Pk，且i<j<k，三个点成关系，Pj点在直线Pi->Pk的上凸部分：k(Pi,Pj)>k(Pj,Pk)，可以发现Pj点是多余的。

若k(Pt,Pj)> k(Pt,Pi)，那么可以看出，Pt点一定要在直线Pi Pj的上方，即1号区域。同理若k(Pt,Pj)> k(Pt,Pk)，那么Pt点一定要在直线PjPk的下方，即的2号区域。

综合上述两种情况，若PtPj的斜率同时大于Pt Pi和PtPk的，Pt点一定要落在两阴影的重叠部分，但这部分显然不满足开始时t>j的假设。于是，Pt落在任何一个合法的位置时，PtPj的斜率要么小于PtPi，要么小于PtPk，即不可能成为最大值，因此Pj点多余，完全可以删去。

这个结论告诉我们，任何一个点Pt的检查集合中，不可能存在一个对最优结果有贡献的上凸点，因此我们可以删去每一个上凸点，剩下的则是一个下凸折线。最后需要在这个下凸折线上找一点与Pt点构成的直线斜率最大——显然这条直线是在与折线相切时斜率最大，如图所示。

斜率优化

下面正式讲斜率优化。

对于一类Dp方程式：f[i]=min(f[j]+a[i]∗b[j]+c[j]+d[i]) a[x],b[x],c[x],d[x]都为x的函数，且a单增。

在 i 时对于最优决策 j : f[i]=f[j] -a[i]∗b[j]+c[j] +d[i]
                   (f[i])=-a[i] ∗ b[j]+ (c[j]+d[i]+f[j])             
                       b    =-k   *     x  +  y   (y=kx+b)

对于任意j可以表示为图像上一点J( b[j],c[j]+d[i]+f[j] )，目的是求出f[i],因为f[i]的值等于y=kx+b直线的截距。

问题转化为：对于一条斜率k = a[i] 的直线（为什么将它作为斜率?），使其至少过一点J(即上文那个任意J点,j<i)，使得该直线的截距最小(就是与y轴焦点的纵坐标值)。那么考虑哪些J点有望成为答案？

因为a[i]单调递增，即斜率逐渐变大。所以可以维护一个下凸包。逐渐去除尾部

而加入的点用上面的方式去除不可能成为最优解的解。

在代码实现“去除尾巴”和“清理头部”中，都是以判断斜率为标准,所以是叫斜率优化

注意到了吗J( b[j],c[j]+d[i]+f[j] )那么每次d[i]都在变化。但在求斜率这个d[i]会被消去故不影响。

回归数学

还记得决策单调性和算斜率的一大堆推导吗？

决策单调性就是证明了d[i]对每次决策没有影响（既然在求i时，j比k要优，那么之后，即在t(t>i)时，j是否也比k要优）。似乎是在证a单增。这步似乎没什么用特别是对a[i]不单增时更没用

而算斜率就是从两个J ( b[j],c[j]+d[i]+f[j] )中算出斜率。
